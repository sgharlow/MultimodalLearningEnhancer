# 📸 Screenshot Capture Guide

**Goal:** Capture 6 professional screenshots for README and DevPost submission

**Time Required:** 10-15 minutes

---

## Screenshot Requirements

**Format:** PNG (preferred) or JPG
**Size:** 1920x1080 or larger (high DPI displays are fine)
**Quality:** Clear, readable text
**Content:** Clean UI, no personal info, professional appearance

---

## Tools

### Option 1: Built-in Screenshot (Recommended)
- **Windows:** `Win + Shift + S` → Select area
- **Mac:** `Cmd + Shift + 4` → Select area
- **Chrome DevTools:** F12 → Cmd/Ctrl + Shift + P → "Capture screenshot"

### Option 2: Browser Extension
- **Awesome Screenshot** (Chrome Web Store)
- **Fireshot** (Chrome Web Store)

### Option 3: Snipping Tool / Screenshot App
- Windows: Snipping Tool
- Mac: Screenshot app (Cmd + Shift + 5)

---

## Screenshot Preparation

### Before Capturing:

1. **Clean Browser**
   - Close unnecessary tabs
   - Hide bookmarks bar (Ctrl/Cmd + Shift + B)
   - Use Incognito/Private mode for clean UI (optional)

2. **Zoom Level**
   - Set browser zoom to 100% (Ctrl/Cmd + 0)
   - Ensures consistent sizing

3. **Window Size**
   - Maximize browser window OR
   - Set to standard size (1920x1080)

4. **DevTools**
   - Close DevTools for cleaner screenshots
   - Only open if showing console output

5. **Widget Position**
   - Drag widget to optimal position
   - Avoid covering important content
   - Ensure entire widget is visible

## Tips for Great Screenshots

### Do:
- ✅ Use real, meaningful content (Wikipedia, MDN)
- ✅ Wait for full rendering
- ✅ Capture clean, uncluttered UI
- ✅ Show the extension working
- ✅ Highlight your Path A features (Cornell Notes!)

### Don't:
- ❌ Include personal information
- ❌ Show error states (unless demonstrating error handling)
- ❌ Use tiny fonts or low resolution
- ❌ Capture while loading
- ❌ Include unnecessary browser UI
